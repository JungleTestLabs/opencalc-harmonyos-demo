## 1 使用说明

### 1.1 列字段定义

| 列名 | 说明 | 填写要求 |
|------|------|----------|
| 一级对象 | 发生故障的对象类别 | 必填 |
| 二级对象 | 故障发生的具体子系统或组件 | 必填 |
| 三级对象 | 具体的故障位置或场景（可选） | 选填 |
| 四级对象 | 进一步细化的故障位置（可选） | 选填 |
| 故障模式 | 故障对象可能出现故障的方式 | 必填 |
| 严酷度 | I级关键/Ⅱ级重要/Ⅲ级一般/Ⅳ级提示 | 选填 |
| 发生频度 | 1-10，数字越大越容易发生 | 选填 |
| 可检测度 | 1-10，数字越大越难检测 | 选填 |
| SIL等级 | 安全完整性等级 | 选填 |
| 故障原因（对象） | 引发故障的对象 | 选填 |
| 故障原因（行为） | 引发故障的具体行为 | 选填 |
| 故障影响（对象） | 受到故障影响的对象 | 选填 |
| 故障影响（详情） | 故障产生的具体影响描述 | 选填 |
| 改进措施 | 针对该故障模式的改进建议 | 选填 |

### 1.2 RPN评分

| 维度 | 范围 | 说明 |
|------|------|------|
| 严重度S | 1-10 | 后果严重程度，10最严重 |
| 发生率O | 1-10 | 发生可能性，10最可能 |
| 探测度D | 1-10 | 探测难度，10最难 |

### 1.3 RPN阈值

| RPN | 风险 | 处理 |
|-----|------|------|
| ≥100 | 高 | 必须改进措施 |
| 60-99 | 中 | 建议改进措施 |
| <60 | 低 | 可接受监控 |

**故障模式 9**: 业务过载


**故障原因（对象）**:
待定（FMEA分析确定对象）

**故障原因（行为）**:
业务量大

**故障影响（对象）**:
待定（FMEA分析确定对象）

**故障影响（详情）**:
系统可能性能下降或失去业务处理能力。

**改进措施**:
确定系统业务过载的瓶颈资源，并对其进行过载的定时监控检测。
检测到过载后，系统应采取一定的保护策略来维持在系统能力范围内正常提供业务。过载控制措施应系统性地根据具体业务的特点考虑，可参考以下几种相关措施（可以结合使用）：
根据敏感因素动态/静态调整业务接入量，保证敏感因素不会因为业务突发而全部被占用导致系统丧失业务处理能力。
应对业务区分优先级，根据用户对不同业务的不同要求制定控制策略，如对高优先级业务不直接丢弃，采用缓存或其他方式，或对高优先级业务的控制力度比低优先级业务小，或者低优先级业务直接丢弃，只保留高优先级业务。
采用合理的队列管理策略和调度策略，在业务过载时能够保证需优先处理消息能够得到及时处理(例如系统内部管理消息)，队列管理、调度策略可以根据不同消息的时延要求及重要性等因素制定，可以与第二条结合使用。
