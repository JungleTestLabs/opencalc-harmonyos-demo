## 1 使用说明

### 1.1 列字段定义

| 列名 | 说明 | 填写要求 |
|------|------|----------|
| 一级对象 | 发生故障的对象类别 | 必填 |
| 二级对象 | 故障发生的具体子系统或组件 | 必填 |
| 三级对象 | 具体的故障位置或场景（可选） | 选填 |
| 四级对象 | 进一步细化的故障位置（可选） | 选填 |
| 故障模式 | 故障对象可能出现故障的方式 | 必填 |
| 严酷度 | I级关键/Ⅱ级重要/Ⅲ级一般/Ⅳ级提示 | 选填 |
| 发生频度 | 1-10，数字越大越容易发生 | 选填 |
| 可检测度 | 1-10，数字越大越难检测 | 选填 |
| SIL等级 | 安全完整性等级 | 选填 |
| 故障原因（对象） | 引发故障的对象 | 选填 |
| 故障原因（行为） | 引发故障的具体行为 | 选填 |
| 故障影响（对象） | 受到故障影响的对象 | 选填 |
| 故障影响（详情） | 故障产生的具体影响描述 | 选填 |
| 改进措施 | 针对该故障模式的改进建议 | 选填 |

### 1.2 RPN评分

| 维度 | 范围 | 说明 |
|------|------|------|
| 严重度S | 1-10 | 后果严重程度，10最严重 |
| 发生率O | 1-10 | 发生可能性，10最可能 |
| 探测度D | 1-10 | 探测难度，10最难 |

### 1.3 RPN阈值

| RPN | 风险 | 处理 |
|-----|------|------|
| ≥100 | 高 | 必须改进措施 |
| 60-99 | 中 | 建议改进措施 |
| <60 | 低 | 可接受监控 |

## 3 故障模式详情

### 3.1 硬件

#### 3.1.1 CPU及外围电路

**故障模式 1**: CPU主频发生变化


**故障原因（对象）**:
待定（FMEA分析确定对象）

**故障原因（行为）**:
配置被改写

**故障影响（对象）**:
待定（FMEA分析确定对象）

**故障影响（详情）**:
系统运算速率发生变化，可能导致系统功耗过大，或系统反应慢

**改进措施**:
部分CPU可通过读主频配置寄存器判断是否与初始配置不一致，检测到不一定可告警提示用户重新配置。

**故障影响（对象）**:
待定（FMEA分析确定对象）

**故障影响（详情）**:
正常CPU占有率过高，系统复位

**改进措施**:
可考虑芯片级冗余（多CPU）或板级冗余设计进行保护。

**故障模式 3**: CPU单核故障(多核场景)

待定（FMEA分析确定对象）

**故障影响（详情）**:

检测到单核故障后，将故障核进行业务隔离，业务由其他核分担。
可采取单核复位或其他措施进行恢复。

#### 3.1.3 存储阵列

**故障模式 1**: ABRT（Aborted,不响应ATA命令）


**故障原因（对象）**:
待定（FMEA分析确定对象）

**故障原因（行为）**:
硬盘出现错误

**故障影响（对象）**:
待定（FMEA分析确定对象）

**故障影响（详情）**:
不响应任何ATA命令

**改进措施**:
接收到硬盘故障时重试，若持续失败，则置盘失效，隔离故障盘，进行更高层级的恢复。
告警，提示人工更换故障磁盘。

---

**故障模式 2**: DF(Device Fault全F坏道，不可读写)


**故障原因（对象）**:
待定（FMEA分析确定对象）

**故障原因（行为）**:
硬盘出现错误

**故障影响（对象）**:
待定（FMEA分析确定对象）

**故障影响（详情）**:
全F坏道。不能提供数据读写，需对硬盘下电

**改进措施**:
SSD支持盘内的RAID，在盘内的同一个raid组内，只要坏道没有对齐，盘均能够主动修复。
如果同一个raid组内2个或多个坏道对齐了，盘自己无法修复对齐的坏道，需要将相关的坏道通过pendinglist报给阵列进行及时修复。
如果是读操作碰到坏道后，pool计算出正确数据，返回给主机，然后在后台进行坏道的修复。

**故障模式 3**: DS（Disk Slow慢盘）


**故障原因（对象）**:
待定（FMEA分析确定对象）

**故障原因（行为）**:
硬盘出现错误

**故障影响（对象）**:
待定（FMEA分析确定对象）

**故障影响（详情）**:
慢盘，顺序写小于15MB/s

**改进措施**:
根据一定时间内的磁盘读写延时与正常平均延时(或者经验值)的对比可判断是否发生慢盘。
判断出慢盘后，进行磁盘隔离或者更高层级的恢复。
告警，提示人工更换故障磁盘。

**故障模式 9**: UNC（read UNCorrectable data坏道，读出错写可修复）


**故障原因（对象）**:
待定（FMEA分析确定对象）

**故障原因（行为）**:
硬盘出现错误

**故障影响（对象）**:
待定（FMEA分析确定对象）

**故障影响（详情）**:
坏道。读出错，写可修复，此故障发生最频繁。逻辑坏道可能由于异常掉电等，发生数据ECC校验错误。物理坏道即物理介质损坏，硬盘固件进行坏块替换。

**改进措施**:
RAID组单盘出现UNC坏道，采用局部重构方式对该LBA地址进行修复。
RAID组降级后若无冗余，成员盘出现UNC坏道，对坏道进行标记，不踢盘。
硬盘双盘同LBA地址出现UNC坏道，对坏道进行标记，不踢盘。
在一段时间内（100s）内，同一盘发生10次校验失败，则置盘即将失效。否则清空计数器并重新计数。
