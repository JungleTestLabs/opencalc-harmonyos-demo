# Skill 优化导向审阅指南

当用户要求“评估 skill”“优化 skill”“根据检视结果改 skill”“分析生成这些文档的 skill 能力”时使用本指南。

目标不是只指出文档缺陷，而是把文档缺陷反推为可修改的 skill 规则、模板、自检或路由问题。

## 反推原则

1. 先确认文档缺陷有证据，再反推 skill 缺陷。
2. 不把单个文档偶发问题直接判定为 skill 系统性缺陷；必须标注置信度。
3. skill 优化建议必须能落到 `SKILL.md`、`references/`、`templates/`、自检步骤或 eval case。
4. 如果缺陷来自输入缺失，应建议 skill 增加输入前置检查，而不是只要求生成更完整内容。
5. 如果缺陷来自事实错误，应建议 skill 增加证据引用、代码仓核验或外部事实复核规则。

## Skill 缺陷标签

| 标签 | 名称 | 判定信号 | 常见修复 |
|------|------|----------|----------|
| S1 | 输入理解问题 | 误判需求类型、忽略用户约束、跳过 blocker | 增加输入解析步骤、阻断条件、澄清触发规则 |
| S2 | 输出结构问题 | 缺章节、缺字段、字段顺序不稳定 | 固化输出模板、必填字段清单、空值处理规则 |
| S3 | 生成约束问题 | 模糊表述、不可验收、没有边界 | 增加可判定规则、验收标准规则、禁止泛化词 |
| S4 | 追溯链路问题 | 上下游引用断裂、决策来源缺失 | 增加 ID 规则、追溯矩阵、决策来源字段 |
| S5 | 事实校验问题 | 编造平台限制、代码事实、标准或 API | 增加证据要求、代码路径引用、需复核标注 |
| S6 | 自检缺失问题 | 同类缺陷可由生成后检查发现但未发现 | 增加生成后 checklist、critic、自测步骤 |
| S7 | 模板缺陷问题 | skill 按模板生成但模板本身缺字段 | 修改模板、增加示例、增加反例 |

## D 维度到 S 标签的默认映射

| 文档维度 | 默认 skill 缺陷标签 |
|----------|---------------------|
| D1 清晰性 | S3 生成约束问题 |
| D2 完整性 | S2 输出结构问题 / S7 模板缺陷问题 |
| D3 一致性 | S4 追溯链路问题 / S6 自检缺失问题 |
| D4 准确性 | S5 事实校验问题 |
| D5 追溯性 | S4 追溯链路问题 |
| D6 可行性 | S1 输入理解问题 / S5 事实校验问题 |
| D7 可验证性 | S3 生成约束问题 / S6 自检缺失问题 |
| D8 可维护性 | S2 输出结构问题 / S7 模板缺陷问题 |

## 置信度

| 置信度 | 使用条件 |
|--------|----------|
| 高 | 同一缺陷在多个位置重复出现，且能对应到明确缺失的 skill 规则或模板字段 |
| 中 | 缺陷证据充分，但无法确认是 skill 规则缺失还是输入缺失 |
| 低 | 只有单点缺陷，或需要查看 skill 源文件后才能确认 |

## 建议修改位置

- `SKILL.md`：补触发条件、工作流、阻断条件、输出原则。
- `references/*.md`：补规则、检查项、事实依据、反例。
- `templates/*.md`：补必填字段、决策来源、追溯字段。
- `evals.json`：补能复现缺陷的评测样例。
