# 国际化界面用户规范

## 概述

本文档定义了终端产品国际化界面的用户规范要求，包括多语言支持、界面适配、本地化等内容。

## 基本要求

### 1. 多语言支持范围

#### 1.1 中国大陆发布

**必需语言**：
- 中文（简体）
- 英文
- 繁体中文（香港）
- 繁体中文（台湾）
- 藏文
- 维文

**验证要求**：
- [ ] 是否支持6种语言切换
- [ ] 语言切换功能是否完整
- [ ] 各语言翻译是否准确
- [ ] 特殊语言（藏文、维文）显示是否正确

#### 1.2 全球发布

**必需内容**：
- 分析是否涉及海外销售/上架
- 是否收集个人信息
- 单框架设计规范-全球化-国际化
- 全球化规范（规范编号 DKBA 10299-2025.05）

**验证要求**：
- [ ] 是否涉及海外发布
- [ ] 如涉及，是否满足全球化规范
- [ ] 个人信息收集是否符合当地法规
- [ ] 是否支持目标市场语言

### 2. 界面适配规范

#### 2.1 文本显示

**基本规则**：
- 文本长度应考虑不同语言差异
- 避免硬编码文本宽度
- 使用自动换行和弹性布局

**验证要求**：
- [ ] 长文本是否会溢出
- [ ] 不同语言文本对齐是否正确
- [ ] 特殊字符（如藏文、维文、阿拉伯文）显示是否正常

#### 2.2 数字和日期格式

**本地化要求**：
- 数字格式：千位分隔符、小数点
- 日期格式：年-月-日、月/日/年等
- 时间格式：12小时制/24小时制
- 货币格式：符号位置、小数位数

**验证要求**：
- [ ] 数字格式是否本地化
- [ ] 日期时间格式是否本地化
- [ ] 货币显示是否正确

#### 2.3 文本方向

**支持的语言方向**：
- 从左到右（LTR）：中文、英文等
- 从右到左（RTL）：阿拉伯文、希伯来文等

**验证要求**：
- [ ] RTL语言界面是否正确翻转
- [ ] 混合语言文本显示是否正确
- [ ] 图标和按钮方向是否适配

### 3. 图标和符号

#### 3.1 文化敏感性

**注意事项**：
- 避免使用具有特定文化含义的图标
- 符号应在不同文化中具有相同含义
- 颜色在不同文化中的含义

**验证要求**：
- [ ] 图标是否具有普遍认知
- [ ] 是否避免文化敏感符号
- [ ] 颜色使用是否合适

#### 3.2 文本与图标

**设计原则**：
- 重要文本不应仅依赖图标
- 提供文本标签辅助理解
- 确保图标+文本组合在不同语言下正常显示

**验证要求**：
- [ ] 是否提供文本标签
- [ ] 图标+文本布局是否合理
- [ ] 不同语言下布局是否一致

### 4. 输入法适配

#### 4.1 文本输入

**支持要求**：
- 支持目标市场主流输入法
- 特殊字符输入支持
- 输入法切换便捷性

**验证要求**：
- [ ] 是否支持目标语言输入法
- [ ] 特殊字符是否可输入
- [ ] 输入法切换是否流畅

#### 4.2 输入验证

**验证规则**：
- 姓名字段支持多语言字符
- 地址字段支持不同格式
- 电话号码格式验证

**验证要求**：
- [ ] 姓名字段是否支持多语言字符
- [ ] 地址字段是否支持不同格式
- [ ] 电话号码验证是否本地化

```
[ ] 帮助文档是否本地化
```

### 6. 常见问题

#### 问题1：文本溢出

**问题**：翻译后文本过长导致界面溢出

**解决方案**：
- 使用弹性布局
- 设置最大文本长度
- 提供文本滚动或截断

#### 问题2：特殊字符显示异常

**问题**：藏文、维文等特殊语言字符显示不正确

**解决方案**：
- 使用系统字体支持
- 确保字符编码正确
- 测试特殊字符渲染

#### 问题3：布局错乱

**问题**：不同语言文本长度差异导致布局不一致

**解决方案**：
- 使用相对布局而非绝对定位
- 确保弹性容器正确设置
- 测试各种语言下的布局

### 8. 自动验证脚本示例

```python
import re

def validate_internationalization(prd_file):
    """验证PRD中的国际化要求"""
    with open(prd_file, 'r', encoding='utf-8') as f:
        content = f.read()

    issues = []

    # 检查是否提到国际化
    if '国际化' not in content and '多语言' not in content:
        issues.append("未提及国际化或多语言支持")

    # 检查中国大陆发布
    if '中国大陆' in content:
        if '6种语言' not in content and '六种语言' not in content:
            issues.append("涉及中国大陆发布但未说明6种语言支持")

    # 检查全球发布
    if '全球' in content or '海外' in content:
        if 'DKBA 10299' not in content:
            issues.append("涉及海外发布但未引用全球化规范DKBA 10299")

    return issues
```
