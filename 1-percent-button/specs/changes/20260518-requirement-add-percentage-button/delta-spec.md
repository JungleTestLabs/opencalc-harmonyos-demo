## 系统背景

### 目的

> 在 OpenCalc HarmonyOS 计算器主界面新增 `%` 百分号按钮，使用户可直接在表达式中输入百分号，并复用已有的智能百分号语义引擎（`a+b%` ≡ `a + a*(b/100)`）。本次变更解决「智能百分号语义已实现但 UI 未暴露」的入口缺失问题，对应用户价值：与主流计算器（Android 计算器、iOS 计算器、Casio 等）保持一致的百分号交互体验。

### 术语与缩写

| 术语/缩写 | 定义 |
| -------- | --- |
| `%` | 百分号字符（U+0025），在本应用中作为运算符使用 |
| 智能百分号 | 上下文相关的百分号语义：`a+b%` 等价于 `a + a*(b/100)`；`a-b%` 等价于 `a - a*(b/100)`；`a*b%` 等价于 `a*(b/100)`；`a/b%` 等价于 `a/(b/100)`；独立 `b%` 等价于 `b/100` |
| 基础模式 | 计算器仅显示基础按钮（数字、四则、AC、退格、等号、括号）的模式 |
| 科学模式 | 计算器额外显示 sin/cos/tan/ln/log/反三角函数/π/e 的模式 |
| ButtonGrid | `CalculatorPage.ets` 中 `@Builder ButtonGrid()` 渲染的按钮网格 |
| `getOp()` | 运算符颜色函数，返回当前主题下的运算符按钮背景色（如 `#B4D2E4`） |

### 关键体验路径（KEP）

- **KEP-001：百分比加成计算**
  - 用户目标：计算 50 元加 10% 服务费的总额
  - 路径（起点→关键步骤→终点）：
    1. 起点：用户已在主界面（基础模式），表达式区为空
    2. 依次点击 `5` `0` `+` `1` `0` `%`

    5. 结果区显示 `55`
  - 体验预期：从首次点击到看到结果 ≤ 2 秒（含动画），无明显卡顿

- **KEP-002：百分比折扣计算**
  - 用户目标：计算 100 元打 8 折后的金额
  - 路径：依次点击 `1` `0` `0` `-` `2` `0` `%` `=`
  - 体验预期：结果显示 `80`

- **KEP-003：纯百分比换算**
  - 用户目标：把 25% 转换为小数
  - 路径：依次点击 `2` `5` `%` `=`

##### 触发事件

##### 主成功路径（基本事件流）

```
1. 用户点击 5 → expression = "5"
2. 用户点击 0 → expression = "50"
3. 用户点击 + → expression = "50+"
4. 用户点击 1 → expression = "50+1"
5. 用户点击 0 → expression = "50+10"
6. 用户点击 % → expression = "50+10%"
7. 用户点击 = → 触发 onEquals()
   7a. Expression.getCleanExpression("50+10%") → "50+50*(10/100)"
   7b. CalcEngine.evaluate(..) → 55
   7c. NumberFormatter.format(55, ..) → "55"
   7d. result = "55"，errorMsg = ""
8. 历史记录新增 1 条
```

##### 扩展路径

```
1a. 用户在 5 之前点击其他键 → 按已有规则处理
6a. 用户连续点击 %% → expression = "50+10%%"，求值时由 Expression 处理为有效形式或标记为语法错误
6b. 用户在 6 后点击 = → 同上
```
