# Proposal — 小费计算器（Tip Calculator）

## 意图分类

| 维度 | 值 |
|------|-----|
| 意图类型 | `requirement-add`（新增功能） |
| 领域 | UI + 纯逻辑计算 |
| 父模块 | 新增独立页面 `TipCalculatorPage` |

## 需求描述

在 OpenCalc 计算器基础上增加一个小费计算器功能：
- 用户输入账单总金额、选择小费比例、输入人数
- 自动计算每人应付金额
- 支持预设小费比例（10%、15%、18%、20%）+ 自定义比例
- 从主页面可导航到小费计算器

## 影响层次

| 层级 | 影响 |
|------|------|
| UI 层 | 新增 TipCalculatorPage.ets（~180 行） |
| 导航层 | 修改 Index.ets（~20 行，增加导航按钮） |
| 计算引擎 | 无修改 |
| 数据模型 | 无修改 |
| 偏好存储 | 无修改 |

## 复杂度评估

- **复杂度**: S 级
- **理由**: 纯 UI + 四则运算，不涉及计算引擎、偏好存储、历史记录。新增 1 个文件 + 1 处导航修改。
- **跳过的阶段**: 架构设计（step7）

## 验收标准

1. 输入金额 100，选择 15% 小费，2 人 → 显示每人 57.50
2. 输入金额 200，选择 18% 小费，4 人 → 显示每人 59.00
3. 选择自定义小费比例 12%，结果正确计算
4. 人数为 0 时显示错误提示
5. 金额为 0 或负数时显示错误提示
6. 从主页面点击导航按钮能进入小费计算器
7. 从小费计算器能返回计算器主页
8. 原有计算器功能不受影响
