# 小费计算器(Tip Calculator) 需求提案

## 1. 需求概述

### 1.1 功能名称
小费计算器(Tip Calculator)

### 1.2 需求类型
新增功能(New Requirement)

### 1.3 关联特性
feat-tip-calculator(本变更首次引入,尚无既存特性目录)

### 1.4 需求描述
在 OpenCalc HarmonyOS 应用中新增一个独立的小费计算器页面。用户从主页点击入口进入,在该页面输入账单金额(两位小数)、就餐人数(正整数,≥1)、选择小费率(预设 10/15/18/20%,默认 15%,支持自定义),系统实时计算并显示**每人应付金额**(保留两位小数)。

### 1.5 使用场景
- 餐厅就餐结束后,几位同事/朋友需要快速决定每人 AA 付多少钱(含小费)。
- 用户希望在不离开 OpenCalc 这一计算工具应用的前提下完成小费 + 分账的常见场景。

### 1.6 预期效果
- 用户在 ≤ 3 步操作内即可获得"人均应付"结果。
- 计算结果实时随输入变化更新,无需额外点击"计算"按钮。
- 不影响 OpenCalc 现有基础计算器功能。

## 4. 需求详述

### 4.1 功能需求

#### FR-1: 主页新增小费计算器入口
**描述**:在 `Index.ets` 主页面增加一个「小费计算器」按钮,点击后跳转到 `TipCalculatorPage`。
**验收条件**:
- 主页可见一个名为「小费计算器」(或英文 "Tip Calculator",取决于现有语言)的按钮/卡片。
- 点击按钮后,应用导航至 TipCalculatorPage,且支持返回主页。
- 不影响主页原有功能与布局。

#### FR-2: 小费计算器页面输入
**描述**:`TipCalculatorPage` 提供三项输入控件:账单金额、就餐人数、小费率(档位选择 + 自定义)。
**验收条件**:
- 金额输入框接受 ≥0 的两位小数数值,非数字字符被阻止或忽略。
- 人数输入框接受 ≥1 的整数。
- 小费率提供 4 个预设档位按钮:10% / 15% / 18% / 20%,默认选中 15%。
- 提供一个「自定义」输入,用户可填入 0-100 之间的数字(支持整数或小数)。
- 输入与档位/自定义切换时,UI 状态保持互斥与一致。

#### FR-3: 实时人均计算
**描述**:任意输入项变化时,实时计算并显示每人应付金额。
**验收条件**:
- 公式:`perPerson = round((amount × (1 + tipPercent/100)) / peopleCount, 2)`(四舍五入到两位小数)。
- 结果在输入变化后 100ms 内更新(响应性)。
- 输入为空、非法或人数为 0 时,人均显示「--」,不报错也不抛异常。
- 显示数值保留两位小数(如 12.50 而非 12.5)。

#### FR-4: 返回与重置
**描述**:支持返回主页;不持久化任何输入。
**验收条件**:
- 页面提供返回按钮或支持系统返回手势。
- 离开并重新进入页面后,所有输入与小费率档位恢复默认状态(金额空、人数空、小费率 15%)。

### 4.2 可靠性需求

#### REQ-DFR-001: 输入容错
**描述**:面对非法输入(空、非数字、超出范围)时,页面应保持 UI 稳定不崩溃,人均结果安全显示「--」。
**关注点**:
- 故障检测:输入解析 NaN 或除数为 0 时识别非法状态。
- 故障恢复:在非法状态下不展示错误数字,仅显示占位符「--」。
- 降级策略:计算异常时,UI 不抛出任何错误弹窗,保持可继续输入。

#### REQ-DFR-002: 数值溢出与极端值
**描述**:防止极端大金额或极端大人数造成显示溢出或精度异常。
**关注点**:
- 金额输入限定上限(例如位数限制为 12 位以内的整数 + 2 位小数)。
- 人数输入限定上限(建议 ≤999)。
- 超出限定时,UI 阻止继续输入或截断到上限。
