## 系统背景

### 目的

OpenCalc HarmonyOS 是一款基础计算器工具应用。本次变更为其新增一个**小费计算器(Tip Calculator)** 子工具,帮助用户在用餐结账等场景下,快速基于账单金额、就餐人数、小费率得出每人应付金额,降低用户跨应用切换的成本,丰富 OpenCalc 的计算场景覆盖能力。

### 术语与缩写

| 术语/缩写 | 定义 |
| ----- | --- |
| TipCalculator | 小费计算器,本次新增的子工具 |
| Per-person | 每人应付金额(含小费,平均分摊) |
| Tip Percent | 小费率,以百分比表示(0-100) |
| Preset Tier | 预设小费档位(10%/15%/18%/20%) |
| ArkUI | HarmonyOS 声明式 UI 框架 |
| ArkTS | HarmonyOS 应用开发语言(TypeScript 超集) |

### 关键体验路径(KEP)

- **KEP-001**:餐后 AA 分账主路径
  - 用户目标:得到每人应付的金额(含小费)。
  - 路径(起点→关键步骤→终点):
    1. 启动 OpenCalc → 主页可见「小费计算器」入口
    2. 点击入口 → 进入小费计算器页面
    3. 输入账单金额 → 输入人数 → (默认 15% 小费率)
    4. 立即看到「每人应付」数值
  - 体验预期(可验证表述):
    - 从主页到看到结果不超过 4 次轻触(点击入口 1 次 + 输入金额 1 次 + 输入人数 1 次,默认小费率无需点击)。
    - 任意输入变更后 100ms 内结果刷新。

##### 简要说明

输入过程中页面不崩溃;输入未完整时人均显示 `--`。

输入合法且齐全后,人均区域显示数值(保留两位小数)。

##### 触发事件
用户在金额或人数输入框中输入或修改字符。

##### 主成功路径

```
1. 用户输入金额(例如 100.00)
2. 用户输入人数(例如 3)
3. 默认小费率 15% 被选中
4. 系统计算:total = 100 × (1 + 0.15) = 115;perPerson = 115 / 3 = 38.333... → 38.33
5. UI 显示「每人应付:38.33」
```

##### 扩展路径

```
2a. 用户在第 2 步前先调整小费率档位 → 按 step 4 公式实时计算
4a. 任一输入为空/0 人 → 显示 `--`,等待补全
```
