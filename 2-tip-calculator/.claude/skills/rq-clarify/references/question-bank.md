# 需求澄清问题库

> rq-clarify 的参考文档。按技术领域组织常见澄清问题，帮助快速生成针对性的提问。

## 通用问题（适用于所有需求类型）

### 范围类
- 这个功能的目标用户是谁？（所有用户 / 特定角色）
- 第一版需要做到什么程度？后续可以迭代的部分是？
- 有没有参考的产品或功能？（竞品对标）
- 这个功能是独立的还是需要与现有功能联动？

### 优先级类
- 这个需求的紧迫程度？（P0 必须 / P1 重要 / P2 一般）
- 与其他正在进行的需求有冲突或依赖吗？

### 兼容性类
- 需要支持哪些设备类型？（手机 / 平板 / 折叠屏）
- 最低支持的 HarmonyOS 版本是？

## 按领域分类

### UI/交互 相关

**布局设计**
- 列表是固定高度还是自适应？
- 大量数据时是否需要懒加载（LazyForEach）？
- 空状态页面如何展示？
- 是否需要骨架屏/loading 效果？

**交互行为**
- 点击/长按/滑动分别触发什么操作？
- 是否需要下拉刷新 / 上拉加载更多？
- 操作后是否需要 toast 提示或弹窗确认？
- 列表项是否支持侧滑操作（删除/编辑）？

**视觉要求**
- 是否需要跟随系统深色模式？
- 有没有设计稿或线框图？
- 图标使用系统图标还是自定义图标？

### 数据存储 相关

**存储选型**
- 数据量大约多少条？是否会持续增长？
- 数据结构简单（键值对）还是复杂（需要关联查询）？
- 是否需要跨应用启动保留数据（持久化）？
- 清除应用数据时是否应该一起清除？

**数据模型**
- 每条数据包含哪些字段？
- 字段的类型和约束？（必填/可选/唯一/默认值）
- 数据之间有没有关联关系？（如：播客→剧集 是一对多）

**数据操作**
- 需要支持哪些查询方式？（按时间/按名称/全文搜索）
- 数据更新频率如何？
- 是否需要批量操作？

### 网络通信 相关

**API 设计**
- 后端 API 是现成的还是需要设计？
- API 的数据格式？（JSON / XML / RSS）
- 认证方式？（无认证 / Token / OAuth）

**网络策略**
- 无网络时的降级方案？（显示缓存 / 提示重试 / 完全不可用）
- 请求频率限制？（防抖/节流/轮询间隔）
- 大文件处理？（分片下载/断点续传）

**错误处理**
- 超时时间设置？
- 重试策略？（自动重试几次 / 用户手动重试）
- 各种 HTTP 错误码的处理方式？

### 媒体播放 相关

**播放能力**
- 支持哪些格式？（MP3 / AAC / M4A / FLAC）
- 是否需要后台播放？
- 是否需要锁屏控制？
- 是否需要蓝牙/耳机按键控制？

**播放控制**
- 播放速度范围？（HarmonyOS 限制 6 档：0.75/1.0/1.25/1.5/1.75/2.0）
- 是否需要定时停止（睡眠定时器）？
- 是否需要播放位置记忆？
- 播放列表的切歌逻辑？（顺序/循环/随机）

**文件管理**
- 播放本地文件时的来源？（沙箱目录 / 用户选择）
- 缓存策略？（全部缓存 / 仅 WiFi / 不缓存）

### 导航路由 相关

**页面结构**
- 应用有几个主 Tab？
- 是否有二级页面？层级有多深？
- 返回逻辑是什么？（返回上一页 / 返回首页）

**数据传递**
- 页面间需要传递什么数据？
- 目标页面是否需要返回结果给源页面？
- 深链接（DeepLink）需求？

### 权限与安全 相关

- 需要哪些系统权限？
- 用户拒绝权限时的降级方案？
- 敏感数据（密码/Token）如何存储？

## Bug 修复专项问题

- 能描述一下复现步骤吗？（从哪个页面开始，做了什么操作）
- 是每次都能复现还是偶发的？
- 在所有设备上都有还是特定设备？
- 是什么时候开始出现的？之前正常吗？
- 有没有错误日志或截图？
- 期望的正确行为是什么？

## 性能优化专项问题

- 当前的性能指标是多少？（加载时间/帧率/内存占用）
- 目标指标是多少？
- 在什么场景下性能问题最明显？（数据量大/动画多/页面切换）
- 有没有做过性能分析？瓶颈在哪？
