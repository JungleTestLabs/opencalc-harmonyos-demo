# 详细设计审计配置

用于 `design`、`delta-design.md` 或实现设计类文档。这是详细设计文档的轻量审计入口。

## 解析

只提取以下内容：
- 接口契约
- 数据模型
- 算法 / 关键流程
- 状态机
- 错误处理
- 并发 / 幂等说明
- 与架构组件的归属关系
- IPD 增量设计信息（设计目标、设计范围、上下文视图、模块设计、接口设计、DFR 追溯矩阵；仅 `delta-design.md` 需要）

如果需要完整解析规则，只打开 `../parsing-rules.md` 中与 `design.md` 解析相关的章节。

## 检查项

优先运行以下检查：

| 检查项 | 维度 | 验证内容 |
|-------|-----------|----------------|
| DS-01 | D1/D2 | 接口是否覆盖输入、输出、错误、前置/后置条件、副作用和并发语义 |
| DS-02 | D2 | 数据模型是否覆盖字段、约束、关系和索引 |
| DS-03 | D1 | 关键流程或算法是否具体到可实现 |
| DS-04 | D2 | 是否定义错误处理、重试和降级路径 |
| DS-05 | D2 | 状态机是否有初始/终止状态和非法转换处理 |
| DS-06 | D5 | 接口是否归属于架构组件 |
| DS-07 | D3 | 术语是否与需求规格和架构设计一致 |
| DS-08 | D4 | API、库和协议是否真实存在且适用 |
| DS-09 | D6 | 复杂度和数据访问是否符合性能预算 |
| DS-10 | D7 | 接口和状态是否可测试 |
| DS-11 | D8 | 设计是否支持兼容演进 |
| DS-12 | D2/D5/D7 | IPD 增量设计是否承接 proposal、需求规格、代码仓理解和 DFR 追溯 |

如果需要详细规则，只打开 `../check-items-design.md`。

## 输出

报告内容：
- 最重要的 3-5 个实现阻塞点
- 准确的接口 / 模型 / 流程位置
- 缺失的契约要素
- 对代码生成或实现的影响
- 具体设计级修复建议
