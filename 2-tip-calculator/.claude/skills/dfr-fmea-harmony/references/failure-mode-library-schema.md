# 故障模式库模板

> 本文件为故障模式库的生成模板，定义标准结构格式，后续新建故障模式库请按此模板整理。

---

## 文件结构

```
故障模式库/
├── 使用说明        # 列字段定义和RPN评分说明
├── 对象索引目录    # 仅列出故障对象（模块/组件/人），不包含故障模式
└── 故障模式详情    # 每条故障模式的完整信息
```

## 1 使用说明

### 1.1 列字段定义

| 列名 | 说明 | 填写要求 |
|------|------|----------|
| 一级对象 | 发生故障的对象类别，如应用服务类、数据存储类等 | 必填 |
| 二级对象 | 故障发生的具体子系统或组件 | 必填 |
| 三级对象 | 具体的故障位置或场景（可选） | 选填 |
| 故障模式 | 故障对象可能出现故障的方式，用标准规范词描述 | 必填 |
| 严酷度 | I级关键/Ⅱ级重要/Ⅲ级一般/Ⅳ级提示 | 选填 |
| 发生频度 | 1-10，数字越大越容易发生 | 选填 |
| 可检测度 | 1-10，数字越大越难检测 | 选填 |
| 故障原因（对象） | 引发故障的对象，全路径用>分隔 | 选填 |
| 故障原因（行为） | 引发故障的具体行为 | 选填 |
| 故障影响（对象） | 受到故障影响的对象 | 选填 |
| 故障影响（详情） | 故障产生的具体影响描述 | 选填 |
| 改进措施 | 针对该故障模式的改进建议 | 选填 |

### 1.2 RPN评分

| 维度 | 范围 | 说明 |
|------|------|------|
| 严重度S | 1-10 | 后果严重程度，10最严重 |
| 发生率O | 1-10 | 发生可能性，10最可能 |
| 探测度D | 1-10 | 探测难度，10最难 |

### 1.3 RPN阈值

| RPN | 风险 | 处理 |
|-----|------|------|
| ≥100 | 高 | 必须改进措施 |
| 60-99 | 中 | 建议改进措施 |
| <60 | 低 | 可接受监控 |

## 编写注意事项

### 对象划分原则

**场景故障模式**（对象为外部因素）:
- 一级对象：按业务领域或系统类型划分（如：应用服务类、数据存储类、用户行为类）
- 二级对象：具体的子系统或组件（如：数据库、缓存、消息队列、用户）
- 三级对象：可选，更细粒度的故障位置
- 典型对象类型：
  - 人：用户、操作人员、维护人员
  - 外部环境：机房、供电、网络
  - 自然环境：温度、湿度、雷电
  - 周边系统：网管、第三方设备
  - 使用场景：WEB交互、命令行操作、业务对接

**功能故障模式**（对象为系统内部元素）:
- 一级对象：按系统架构层次划分（如：接入层、处理层、存储层）
- 二级对象：具体的架构元素（如：API接口、数据库连接、缓存组件）
- 三级对象：可选，更细粒度的模块或部件
- 典型对象类型：
  - 系统组件：服务进程、后台任务、定时器
  - 接口：API接口、内部消息、RPC调用
  - 模块：业务逻辑模块、配置模块、日志模块
  - 部件：CPU、内存、磁盘、网络卡

### 对象与故障模式的区别
- **对象**：会发生故障的模块、组件、人（如：数据库、WEB应用、用户）
- **故障模式**：具体的故障表现（如：连接池耗尽、页面加载超时、登录失败）

### 内容要求
- 故障模式应具有通用性和代表性
- 故障原因和改进措施应具体可执行
- 保持内容简洁，避免冗余
- 同一类型故障模式库内的对象应避免重复
