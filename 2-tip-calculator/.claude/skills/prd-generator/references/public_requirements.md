# 公共需求/能力参考规则

本文档定义终端V2.1 PRD中公共需求/能力交互影响分析的验证规则。

## 公共需求分类

### 1. 国际化

#### 1.1 中国大陆

**必需内容**：
- 发文：《关于终端平台及产品多语言预置策略要求》
- 6种语言支持：中文/英文/繁体中文/藏文/维文

**验证**：
- [ ] 是否涉及中国大陆发布
- [ ] 如涉及，是否支持6种语言
- [ ] 语言切换功能是否完整

#### 1.2 全球

**必需内容**：

**验证**：
- [ ] 是否涉及海外发布

## 常见问题

### 问题1：公共需求分析不完整

**问题**：只标记"涉及/不涉及"，没有详细说明

**影响**：开发团队无法明确需求边界

**修复**：为每个涉及的公共需求添加详细说明，包括具体功能和实现要求

### 问题2：受影响功能分析缺失

**问题**：整体分析标记"涉及"，但没有列出具体功能

**影响**：无法确定哪些功能需要支持该能力

**修复**：在"受影响的相关功能分析"章节详细列出每个功能的支持情况

### 问题3：竞品分析不足

**问题**：竞品分析过于简单，没有对比维度

**影响**：无法明确产品差异化优势

**修复**：使用表格形式，从多个维度（功能、体验、性能等）对比竞品

## 自动验证命令

### 使用 grep 检查公共需求章节

```bash
# 检查公共需求章节
grep -E "(国际化|安全隐私|全局业务|无障碍|适老化)" prd.md

# 检查涉及标记
grep -E "(涉及|不涉及)" prd.md | head -20

# 检查规范编号
grep -E "DKBA [0-9]+" prd.md
```

### 使用 Python 解析器

```python
import re

def validate_public_requirements(prd_file):
    with open(prd_file, 'r', encoding='utf-8') as f:
        content = f.read()

    # 检查必需的公共需求类别
    required_categories = [
        "国际化", "安全隐私", "无障碍", "适老化",
        "克隆备份", "旋转屏", "分屏", "统一交互"
    ]

    missing = []
    for category in required_categories:
        if category not in content:
            missing.append(category)

    return missing
```
