# 未成年人保护规范

本文档定义 PRD 中未成年人保护相关的规范要求和验证规则。

## 背景与法律依据

### 监管趋势
- 教育领域双减政策发布
- 游戏领域实名限时管控加严
- 音视频直播社交领域防沉迷措施推行
- 未成年人保护监管力度持续加强

### 法律框架
- **《未成年人网络保护条例》**：2024年1月1日实施，合计7章60条
- 上位法：《个人信息保护法》、《未成年人保护法》、《网络安全法》
- 是对《未保法》网络保护章节的细化

### 法定年龄分段
| 类别 | 年龄范围 | 说明 |
|------|---------|------|
| 未成年人 | 0~17周岁 | 未满十八周岁的公民 |
| 儿童 | 0~14周岁 | 未满十四周岁的公民 |

### 非隐私合规领域
- 未成年人模式/内容分层分级/防沉迷等
- 由CBG法务部/合规部牵头识别风险
- 不在隐私合规风险识别范围

## 核心要求

### 1. 商业营销限制

**条款**：第二十四条
- 不得通过自动化决策方式向未成年人进行商业营销
- 不得提供基本功能模式

**重点解读**：
- 针对未成年人不能提供个性化广告/直接营销
- 个性化推荐未明确禁止

**应对策略**：

| 年龄段 | 个性化广告 | 直接营销 | 个性化推荐 |
|--------|-----------|----------|-----------|
| 儿童（0~14） | 全面禁止 | 全面禁止 | 默认关闭，开启需监护人同意 |
| 未成年人（15~17） | 禁止 | 禁止 | 暂时保持现状 |

**基线要求**：
- 合入隐私合规基线
- 产品需根据年龄段进行差异化处理

### 2. 个人信息最小必要

**条款**：第三十二条
- 严格遵守国家网信部门和有关部门关于网络产品和服务必要个人信息范围的规定
- 不得强制要求未成年人或其监护人同意非必要的个人信息处理行为
- 不得因为不同意处理非必要个人信息或撤回同意，拒绝未成年人使用其基本功能服务

**重点解读**：
- 同《个保法》第16条，未成年人也需要满足最小必要规定
- 参考《常见类型移动互联网应用程序必要个人信息范围规定》
- 如果服务提供的均是基本功能服务，不需要按照基本功能模式处理

**应对策略**：

#### 方案一：统一基本功能模式
1. 基本功能模式不区分成人/未成年人用户
2. 首次启动欢迎页点击"取消"都提供基本功能模式
3. 收集个人信息满足《必要信息范围规定》要求
4. **内容分层分级要求**：不收集任何个人信息，也无法判断是否儿童前提下，提供基本功能的内容不强制做内容分层分级

#### 方案二：动态同意模式
1. 进入APP时，只能使用基本功能
2. 用户在使用非基本功能前，动态获得用户的同意

#### 单框架方案：全局儿童保护模式
- 当家长在设备上开启未成年人模式后
- 应用通过Account Kit提供的未成年人模式相关能力
- 与系统联动，主动切换至未成年人模式
- 根据未成年人的年龄段信息提供适龄内容

## PRD 检查清单

### 功能设计检查
- [ ] 是否涉及未成年人用户
- [ ] 是否收集未成年人个人信息
- [ ] 是否提供个性化广告/推荐
- [ ] 是否实现基本功能模式
- [ ] 内容是否进行适龄分级

### 隐私合规检查
- [ ] 隐私声明中是否明确未成年人保护条款
- [ ] 是否获得监护人同意（针对儿童）
- [ ] 个人信息收集是否满足最小必要原则
- [ ] 是否提供未成年人模式

### 技术实现检查
- [ ] 是否集成Account Kit未成年人模式能力
- [ ] 是否申请必要的权限
- [ ] 是否实现年龄段识别
- [ ] 是否实现内容分级展示

### 用户体验检查
- [ ] 未成年人模式切换是否流畅
- [ ] 适龄内容展示是否准确
- [ ] 监护人控制是否完整
- [ ] 退出未成年人模式是否安全

## 常见问题

### 问题1：未区分年龄段
**问题**：所有用户使用相同的处理方式

**影响**：可能违反相关法规要求

**修复**：
- 实现年龄段识别
- 根据不同年龄段提供差异化处理
- 针对儿童（0~14）实施更严格的保护措施

### 问题2：强制收集非必要信息
**问题**：未成年人使用基本功能时，要求同意非必要信息收集

**影响**：违反最小必要原则

**修复**：
- 实现基本功能模式
- 基本功能不收集非必要信息
- 非基本功能使用前动态获取同意

### 问题3：内容未分级
**问题**：向未成年人展示不适合的内容

**影响**：违反内容规范要求

**修复**：
- 实现内容分级系统
- 根据年龄段过滤内容
- 提供适龄内容展示

### 问题4：缺少监护人控制
**问题**：儿童使用应用时没有监护人控制机制

**影响**：无法保护儿童权益

**修复**：
- 实现监护人认证
- 提供家长控制功能
- 记录儿童使用情况

## 测试用例

### TestQI_001：未成年人年龄段识别
1. 检查应用是否能正确识别未成年人年龄段
2. 验证不同年龄段的处理策略是否正确

**预期结果**：
- 能正确识别儿童（0~14）和未成年人（15~17）
- 不同年龄段使用不同的处理方式

### TestQI_002：基本功能模式
1. 检查是否提供基本功能模式
2. 验证基本功能是否满足最小必要原则

**预期结果**：
- 提供基本功能模式
- 基本功能不收集非必要信息

### TestQI_003：个性化广告/推荐
1. 检查儿童是否收到个性化广告
2. 检查未成年人是否收到直接营销

**预期结果**：
- 儿童不接收个性化广告和直接营销
- 未成年人不接收直接营销

### TestQI_004：监护人同意
1. 检查儿童使用功能时是否获取监护人同意
2. 验证同意流程是否完整

**预期结果**：
- 儿童使用敏感功能前获取监护人同意
- 同意流程清晰可理解

### TestQI_005：内容分级
1. 检查内容是否进行适龄分级
2. 验证不同年龄段看到的内容是否不同

**预期结果**：
- 内容有适龄分级标识
- 不同年龄段看到适合的内容

## 示例代码

### 检查是否需要未成年人保护

```python
def check_minor_protection_required(prd_content):
    """
    检查PRD是否需要未成年人保护
    """
    indicators = [
        "儿童",
        "未成年人",
        "学生",
        "教育",
        "游戏",
        "社交",
        "直播",
        "短视频"
    ]
    
    has_minor_users = any(indicator in prd_content for indicator in indicators)
    collects_personal_info = "个人信息" in prd_content or "收集" in prd_content
    
    if has_minor_users and collects_personal_info:
        return {
            "required": True,
            "reason": "应用涉及未成年人用户并收集个人信息",
            "requirements": [
                "实现年龄段识别",
                "提供基本功能模式",
                "禁止个性化广告和直接营销",
                "内容适龄分级",
                "监护人控制"
            ]
        }
    
    return {"required": False}
```

### 验证未成年人保护条款

```python
def validate_minor_protection(prd_content):
    """
    验证PRD中未成年人保护条款的完整性
    """
    checks = {
        "年龄段识别": "年龄段" in prd_content or "年龄识别" in prd_content,
        "基本功能模式": "基本功能" in prd_content,
        "个性化广告": "个性化广告" in prd_content,
        "内容分级": "内容分级" in prd_content or "适龄" in prd_content,
        "监护人同意": "监护人" in prd_content or "家长" in prd_content
    }
    
    missing = [k for k, v in checks.items() if not v]
    
    return {
        "complete": len(missing) == 0,
        "missing": missing,
        "score": len([v for v in checks.values() if v]),
        "max_score": len(checks)
    }
```
