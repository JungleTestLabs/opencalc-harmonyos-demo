# 收集个人信息清单规范

本文档定义基于监管要求的《收集个人信息清单》和《已收集个人信息清单》的验证规则，适用于 PRD 文档中相关章节的规范性检查。

## 概述

《收集个人信息清单》和《已收集个人信息清单》是隐私合规的重要组成部分，旨在向用户透明展示应用收集的个人信息情况。

### 适用范围

#### 有桌面图标的应用

| 数据类型 | 是否需要提供《收集个人信息清单》 |
|---------|--------------------------------|
| 个人数据上云 | ✅ 提供 |
| 个人信息不上云，但涉及工信部关注的个人信息 | ✅ 提供 |
| 个人信息不上云，涉及敏感个人信息 | ✅ 提供 |
| 个人信息不上云，不涉及上述信息 | ❌ 不需要 |

#### 无桌面图标的应用

| 上架情况 | 数据类型 | 是否需要提供《收集个人信息清单》 |
|---------|---------|--------------------------------|
| 不上架应用市场 | 任意 | ❌ 不需要 |
| 上架应用市场 | 个人数据上云 | ✅ 提供 |
| 上架应用市场 | 不上云，但涉及工信部关注的个人信息 | ✅ 提供 |
| 上架应用市场 | 不上云，涉及敏感个人信息 | ✅ 提供 |
| 上架应用市场 | 不上云，不涉及上述信息 | ❌ 不需要 |

#### 无图标无访问入口的应用

❌ 无需提供

### 特殊平台要求

以下平台需在隐私声明中提供《收集个人信息清单》链接，通过 webview 或浏览器打开：

- 小程序
- 元服务
- Web应用
- 操作系统
- 智能手表
- 儿童手表
- 运动手表/手环
- PC
- 中控屏
- 智能音箱
- IoT设备
- 智能协作终端

## 《收集个人信息清单》规范

### 表格模板

| 字段 | 字段说明 | 是否必填 |
|------|---------|---------|
| 业务场景 | 按照隐私声明中的场景介绍 | ✅ 是 |
| 收集目的 | 明确收集个人信息用于哪个具体功能 | ✅ 是 |
| 个人信息类型 | 参考"个人信息描述规则"描述 | ✅ 是 |
| 收集方式 | 用户提供/App收集/从合作伙伴获取 | ✅ 是 |
| 个人信息字段 | 参考个人信息描述规则 | ✅ 是 |

### 收集方式规范

工具化生成时，统一使用以下描述：
- 用户提供
- App收集
- 从合作伙伴获取

### 敏感个人信息突出显示

如收集敏感个人信息，以下字段需**加粗**显示：
- 个人信息字段
- 收集目的

### 展示要求

- 在产品一级或二级菜单添加《收集个人信息清单》入口
- 典型路径：我的 -> 设置页面的"了解我们如何使用您的数据"栏目
- 在产品隐私声明中增加《收集个人信息清单》访问链接

### PRD 文档要求

在 PRD 文档中，需要明确：

1. **是否需要提供《收集个人信息清单》**
   - 根据应用类型和数据处理情况判断
   - 说明判断依据

2. **个人信息收集情况**
   - 列出所有收集的个人信息类型
   - 说明收集目的和使用场景
   - 说明收集方式

3. **敏感个人信息**
   - 标注敏感个人信息
   - 说明敏感个人信息处理方式

4. **展示方案**
   - 说明清单的展示位置
   - 说明清单的访问路径

## 《已收集个人信息清单》规范

### 强制要求产品

以下产品必须提供《已收集个人信息清单》：

**终端云：**
- 浏览器
- 应用市场
- 华为视频
- 华为音乐
- 华为阅读
- 游戏中心
- 教育中心
- 华为主题
- 智慧助手今天
- 华为钱包

**软件部：**
- 运动健康
- 畅连
- 智慧生活

**电商：**
- 华为商城

**MSS：**
- 我的华为

其他产品可选执行。

### 个人信息范围

#### 需要覆盖的个人信息

1. **时间范围**：原则上展示1年以内的已收集的全部个人信息
2. **排除范围**（暂可不包含）：
   - 用户凭证、密码、临时文件类个人信息
   - 华为内部产生的用户无感知个人信息（标签、内部ID等）
   - 通过HASDK途径上报的用于运维运营的数据

#### 仅端侧处理的个人信息

- 有用户界面：建议通过跳转方式实现
- 无用户界面：描述为"已收集，仅端侧处理"

### 个人信息分类

优先参考终端BG个人数据分级分类保护规范的个人信息分类方式，业务可根据实际情况调整。

### 展示内容

#### 必需内容

| 内容项 | 说明 |
|-------|------|
| 使用目的 | 说明个人信息的用途 |
| 使用场景 | 说明个人信息使用的具体场景 |
| 收集情况 | 已收集/未收集 |
| 信息内容 | 具体的个人信息内容 |

#### 特殊处理要求

| 信息类型 | 处理方式 |
|---------|---------|
| 有用户界面的信息 | 通过跳转方式实现，不需要展示时间跨度 |
| 设备标识符 | 根据时间维度展示"已收集/未收集"，不展示条数 |
| 云侧获取的信息（IP地址、位置信息等） | 根据时间维度展示内容信息 |
| 可识别身份的号码（手机号、身份证号等） | 采用脱敏方式展示（如*），不展示完整内容 |
| 第三方SDK采集的信息 | 收集情况按"已收集"展示，信息内容可暂不展示 |
| APP收集后共享给第三方SDK的信息 | 按展示要求完整展示 |

### 时间跨度选择

至少展示1年，并支持以下时间间隔供用户选择性查看：

- 7天（默认）
- 1个月
- 3个月
- 6个月
- 1年

### 展示要求

1. **名称变更**：《收集个人信息清单》变更为《隐私声明摘要》
2. **访问入口**：在产品一级或二级菜单中添加入口
3. **隐私声明链接**：提供《已收集个人信息清单》访问链接

### PRD 文档要求

在 PRD 文档中，需要明确：

1. **是否需要提供《已收集个人信息清单》**
   - 根据产品类型判断
   - 说明判断依据

2. **个人信息展示方案**
   - 说明展示的内容范围
   - 说明个人信息分类方式
   - 说明特殊信息的处理方式

3. **时间跨度方案**
   - 说明支持的时间间隔
   - 说明默认选择的时间跨度

4. **展示位置**
   - 说明清单的展示入口
   - 说明访问路径

## 常见问题

### 问题1：《收集个人信息清单》命名错误

**示例**：
- ❌ 《已收集个人信息清单》
- ✅ 《收集个人信息清单》

**影响**：不符合监管要求

**修复**：确保使用正确的名称

### 问题2：清单入口位置不符合要求

**问题**：清单入口不在产品一级或二级菜单

**影响**：用户无法方便查询

**修复**：将清单入口放置在产品一级或二级菜单

### 问题3：个人信息未在清单中描述

**问题**：应用收集了个人信息，但未在清单中描述

**影响**：不符合隐私合规要求

**修复**：确保所有收集的个人信息都在清单中描述

### 问题4：敏感个人信息未突出显示

**问题**：敏感个人信息的字段未加粗

**影响**：用户无法识别敏感信息

**修复**：将敏感个人信息的字段和收集目的加粗显示
