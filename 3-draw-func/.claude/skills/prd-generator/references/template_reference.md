# PRD 模板参考

本文档提供终端V2.1 PRD模板的详细参考，用于生成和分析PRD文档。

## 模板结构概览

```

├── 文档信息
├── 1. 文档变更记录
├── 2. 产品需求
│   ├── 2.1 需求来源
│   ├── 2.2 需求背景及业务定位
│   ├── 2.3 需求价值分析
│   ├── 2.4 需求适用范围分析
│   │   ├── 产品品类分析
│   │   └── 发布范围分析

│   ├── 2.6 竞品分析（可选）
│   └── 2.7 需求描述

│       ├── 2.7.3 业务流程
```

## 各章节详细说明

### 文档信息

**必需字段**：
- 文档名称
- 版本号（VX.X格式）
- 创建日期
- 最后更新
- 文档状态（草稿/评审中/已批准）

### 1. 文档变更记录

**必需字段**：
- 版本
- 日期
- 修改人
- 修改内容
- 备注

### 2.1 需求来源

**内容要求**：
- 需求提出部门
- 接口人信息
- 需求分类（运营需求/优化需求/用户反馈需求/规划需求）

### 2.2 需求背景及业务定位

**内容要求**：
- 需求的原始诉求
- 需要解决的问题
- 期望达到的目标

### 2.3 需求价值分析

**内容要求**：
- KPI提升
- 用户体验改善
- 用户满意度提升
- 减少运营工作量

### 2.4 需求适用范围分析

#### 产品品类分析

| 产品品类 | 说明 |
|---------|------|
| 手机直板机 | 涉及/不涉及 |
| 手机折叠机（阔折叠、双折叠、三折叠等） | 涉及/不涉及 |
| 平板 | 涉及/不涉及 |
| PC | 涉及/不涉及 |
| 智慧屏 | 涉及/不涉及 |
| 座舱 | 涉及/不涉及 |
| 手表 | 涉及/不涉及 |

#### 发布范围分析

| 发布范围 | 说明 |
|---------|------|
| 仅中国区 | 涉及/不涉及 |
| 亚非拉 | 涉及/不涉及 |
| 欧洲 | 涉及/不涉及 |

### 2.5 目标用户分析（可选）

**内容结构**：
- 用户基本信息（年龄、职业、收入等）
- 产品/服务使用情况及使用经验
- 动机与关注点
- 需求及痛点
- 用户原声

### 2.6 竞品分析（可选）

**分析维度**：
- 典型场景/规格
- HW（华为）方案
- 竞品1、竞品2方案
- 对比分析

**输出内容**：
- 优劣势总结
- 产品/特性竞争力构建计划

### 2.7.1 需求参与者（角色）

**常见角色**：
- 最终用户
- 管理员
- 运营人员

### 2.7.2 前置条件

**检查项**：
- 设备要求
- 系统版本要求
- 权限要求
- 网络要求

### 2.7.3 业务流程

**流程类型**：
- 正常业务流程
- 异常业务流程

**推荐图示**：
- 功能结构图
- 信息结构图
- 任务流程图

### 2.7.4 业务功能描述

**表格字段**：
- 标题
- IR需求编号
- 描述
- 功能名称
- 功能编码
- 备注（界面示意图）

### 2.7.5 关键体验路径（KEP）

**表格字段**：
- 阶段
- 场景类型
- 涉及Actor
- KEP描述
- 场景名称
- 场景编号
- 用例名称
- 用例编号
- IR需求编号

### 2.7.6 关键体验指标（KEI）

**表格字段**：
- 评估体系
- 维度
- 评估类型
- 是否必选
- 评估标准
- IR需求编号
- 功能名称
- 功能编码

### 2.7.7 公共需求/能力交互影响分析

#### 整体分析（必选）

**检查类别**：

##### 国际化
- 中国大陆（6种语言：中/英/港/台/藏/维）
- 全球（海外销售/上架，个人信息收集）

##### 安全隐私
- 儿童账号
- 已收集个人信息清单上报
- 隐私空间
- 停止服务

##### 全局业务
- 无障碍（屏幕朗读、走焦）
- 适老化
- 退账号
- 注销账号
- OOBE
- 克隆备份
- 旋转屏
- 分屏
- 外屏适配
- 自由多窗
- 双升单
- 三方应用关联启动管理
- 单回退双
- 小艺问答操控
- 统一交互（长按/横滑/滑动多选/点击回顶/拖拽等）

#### 受影响的相关功能分析（可选）

**表格字段**：
- 公共需求/能力名称
- 功能点
- 是否支持
- 说明

### 2.7.8 运营策略

## 模板使用要点

### 必填章节

以下章节为必填项：
- 文档信息
- 文档变更记录
- 2.1 需求来源
- 2.2 需求背景及业务定位
- 2.3 需求价值分析
- 2.4 需求适用范围分析
- 2.7 需求描述
  - 2.7.1 需求参与者
  - 2.7.2 前置条件
  - 2.7.3 业务流程
  - 2.7.4 业务功能描述
  - 2.7.5 KEP
  - 2.7.7 公共需求/能力交互影响分析（整体分析）
- 4. 非功能需求

### 可选章节

以下章节为可选项：
- 2.5 目标用户分析
- 2.6 竞品分析
- 2.7.7 公共需求/能力交互影响分析（受影响的相关功能分析）
- 2.7.8 运营策略

### 特殊注意事项

1. **折叠机适配**：涉及折叠机时，需要考虑阔折叠、双折叠、三折叠等差异化的业务流程和KEP/KEI

2. **版本号格式**：使用 VX.X 格式（如 V2.1）

3. **文档状态**：明确标注文档状态（草稿/评审中/已批准）

4. **KEP编号**：遵循场景编号和用例编号规范

5. **公共需求分析**：整体分析为必选，受影响的相关功能分析为可选

6. **运营策略**：至少应包含端内运营和门店体验及演示方案
