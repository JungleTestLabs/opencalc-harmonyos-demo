# 架构设计审计配置

用于 `arch`、`new-arch.md` 或架构类文档。这是架构文档的轻量审计入口。

## 解析

只提取以下内容：
- 架构目标和边界
- 组件及职责
- 依赖关系和数据/控制流
- 技术选型及理由
- NFR 策略
- 部署与可运维性假设
- 风险和权衡
- IPD 架构变更信息（变更原因、变更点、变更后架构、兼容性说明、迁移方案；仅 `new-arch.md` 需要）

如果需要完整解析规则，只打开 `../parsing-rules.md` 中与 `arch.md` 解析相关的章节。

## 检查项

优先运行以下检查：

| 检查项 | 维度 | 验证内容 |
|-------|-----------|----------------|
| AR-01 | D1 | 组件职责是否具体 |
| AR-02 | D2 | 技术栈选型是否包含名称、版本/类别和理由 |
| AR-03 | D5 | 组件是否可追溯到需求 |
| AR-04 | D2/D5 | NFR 是否有架构策略承接 |
| AR-05 | D3 | 术语是否与上游文档一致 |
| AR-06 | D3 | 依赖是否已定义且无环 |
| AR-08 | D4 | 技术/协议是否真实存在且适合场景 |
| AR-09 | D6 | 选型是否能满足目标约束 |
| AR-10 | D7 | 是否具备测试、可观测性和故障注入条件 |
| AR-11 | D8 | 架构是否支持后续演进 |
| AR-12 | D2/D5/D8 | IPD 架构变更是否说明原因、影响、兼容性和迁移策略 |

如果需要详细规则，只打开 `../check-items-arch.md`。

## 输出

报告内容：
- 最重要的 3-5 个架构风险或缺口
- 受影响的组件 / 流程
- 证据
- 对实现的影响
- 具体架构级修复建议
