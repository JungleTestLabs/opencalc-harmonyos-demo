# 需求规格审计配置

用于 `spec`、`delta-spec.md` 或需求类文档。这是需求文档的轻量审计入口。

## 解析

只提取以下内容：
- 范围和非目标
- 参与者 / 角色
- 功能需求
- 非功能需求
- 约束
- 验收标准 / 场景
- 需求 ID 和引用关系
- IPD 增量信息（变更概述、关联特性、DFR 约束、场景分析、用例分析、影响分析；仅 `delta-spec.md` 需要）

如果需要完整解析规则，只打开 `../parsing-rules.md` 中与 `spec.md` 解析相关的章节。

## 检查项

优先运行以下检查：

| 检查项 | 维度 | 验证内容 |
|-------|-----------|----------------|
| SP-01 | D1 | 每条需求是否可判定、单一且不含模糊表述 |
| SP-02 | D2 | 核心需求类别是否覆盖 |
| SP-03 | D5 | ID 和引用是否唯一且有效 |
| SP-04 | D3 | 术语是否一致 |
| SP-05 | D4 | 外部事实、标准和约束是否准确 |
| SP-06 | D6 | NFR 与资源/时间约束是否可行 |
| SP-07 | D7 | 需求是否可测试或可验收 |
| SP-08 | D2/D5/D7 | IPD 增量需求规格是否包含变更链路、场景/用例和影响分析 |

如果需要详细规则，只打开 `../check-items-spec.md`。

## 输出

报告内容：
- 最重要的 3-5 条发现项
- 严重程度
- 准确位置和证据
- 具体改写建议
- 用户要求评分时，才输出评分摘要
