# 功能通用故障模式库

## 概述

本文件包含系统功能的通用故障模式库，适用于各类功能FMEA分析。

**故障模式编号规则**：A开头（A1-A7）

---

## 功能故障模式

| 故障模式编号 | 故障模式 | 描述 | 故障影响 | 改进措施 |
|------------|---------|------|---------|---------|
| A1 | 间歇性功能 | 如操作随机启动-停止-启动 | 业务时好时坏 | 支持业务重试，检测到故障后业务自动切换 |
| A2 | 部分功能 | 如性能损失 | 部分功能不可用 | 功能降级使用，业务最小系统设计等机制 |
| A3 | 功能超范围 | 如操作超出可接受阈值 | 业务异常 | 支持功能的输出检测和安全保护机制 |
| A4 | 非预期功能 | 如在错误的时间操作，意外方向、不相等性能 | 业务异常 | 功能输出结果要增加阈值判断，避免出现严重后果。如果和期望不符则记录现场信息，辅助故障定界定位 |
| A5 | 功能延迟 | 如非预期时间间隔后操作 | 业务响应变慢或超时 | 增加业务KPI检测；增加随路检测，如信令跟踪、服务调用链跟踪等 |
| A6 | 功能丧失 | 如无法操作、突然失效 | 业务不可用 | 功能的故障检测、故障隔离自愈机制 |
| A7 | 功能退化 | 如性能随时间损失/衰退 | 部分功能的性能变差，响应变慢或超时 | 增加业务KPI检测，增加随路检测，如信令跟踪、服务调用链跟踪等 |

---

## 使用说明

在进行功能FMEA分析时，根据待分析系统元素的功能，匹配上述故障模式：

1. **功能丧失（A6）**：当功能完全无法执行时匹配
2. **功能延迟（A5）**：当功能执行时间超出预期时匹配
3. **功能退化（A7）**：当功能性能随时间下降时匹配
4. **间歇性功能（A1）**：当功能时好时坏时匹配
5. **部分功能（A2）**：当功能部分可用时匹配
6. **功能超范围（A3）**：当功能输出超出可接受范围时匹配
7. **非预期功能（A4）**：当功能执行结果不符合预期时匹配

**实例化要求**：在实际应用中，需将故障模式名称和描述实例化，包含具体待分析的系统元素名称。
