# 消息通用故障模式库

## 概述

本文件包含消息的通用故障模式库，适用于各系统间消息交互的FMEA分析。

**故障模式编号规则**：B开头（B1-B6）

---

## 消息故障模式

| 故障模式编号 | 故障模式 | 故障原因 | 故障影响 | 改进措施 |
|------------|---------|---------|---------|---------|
| B1 | 报文丢失或超时 | 业务量大、网络性能不佳等原因可能导致业务消息丢失或超时 | 业务性能下降，严重时可能造成业务中断 | 业务模块（包括同单板的）之间的消息传递，具有可靠性机制，例如：通过序列号检测消息丢失，记录消息丢失，消息丢失后重传等 |
| B2 | 报文超大 | 对端设备或模块bug | 可能导致接收方异常 | 业务模块（包括同单板的）之间的消息需要有消息可靠性机制，例如：对自身不能处理的超大或超长消息有检测规避机制 |
| B3 | 报文重复 | 业务量大、网络性能不佳等原因可能导致业务消息丢失或超时 | 可能导致接收方异常 | 业务模块（包括同单板的）之间的消息需要有消息可靠性机制，例如：能检测到重复消息，并丢弃 |
| B4 | 报文错序 | 业务量很大、网络性能不佳或闪断等原因可能导致错序 | 可能导致接收方异常 | 业务模块对业务消息具备必要的检查和容错措施，如检测到消息错序，能够对其进行重新排序和正确的业务处理 |
| B5 | 报文错误或非法 | 消息传输过程中被篡改 | 可能导致接收方异常 | 业务模块（包括同单板的）之间的消息需要有消息可靠性机制，例如：通过校验检测消息错误 |
| B6 | 业务过载 | 业务量大 | 系统可能性能下降或失去业务处理能力 | 过载时系统不宕机，100倍业务过载保持设计容量90%处理能力；过载时保障高优先级业务运行，高优先级业务可抢占低优先级业务；支持网络级/跨层过载控制协同、自适应能力 |

---

## 使用说明

在进行功能FMEA分析时，根据待分析系统元素的交互信号，匹配上述故障模式：

1. **报文丢失或超时（B1）**：当消息在传输过程中丢失或响应超时时应匹配
2. **报文超大（B2）**：当接收到的消息超出处理能力时应匹配
3. **报文重复（B3）**：当同一消息重复接收时应匹配
4. **报文错序（B4）**：当消息接收顺序与发送顺序不一致时应匹配
5. **报文错误或非法（B5）**：当消息内容错误或格式非法时应匹配
6. **业务过载（B6）**：当系统处理的业务量超出承载能力时应匹配

**实例化要求**：在实际应用中，需将故障模式名称和描述实例化，包含具体待分析的系统元素名称和具体的交互信号。
